<Note>
**Where does `clip_id` come from?**

A `clip_id` identifies any audio clip in the system. You can obtain one from:

| Source | How to get it |
|--------|---------------|
| [Generate Music](/api-reference/music/generate) | `results[].id` in the completed task response |
| [Cover](/api-reference/music/cover) | `results[].id` in the completed task response |
| [Remaster](/api-reference/music/remaster) | `results[].id` in the completed task response |
| [Add Vocals](/api-reference/music/add-vocals) | `results[].id` in the completed task response |
| [Add Instrumental](/api-reference/music/add-instrumental) | `results[].id` in the completed task response |
| [Sample / Humming](/api-reference/music/sample) | `results[].id` in the completed task response |
| [Upload Audio](/api-reference/music/upload) | `data.clipId` (synchronous, returned immediately) |
| [Concat](/api-reference/music/concat) | `data.id` (synchronous, returned immediately) |
| [List Music / Get Music](/api-reference/music/get) | `id` field on each clip object |

For async generation endpoints, poll [Get Task](/api-reference/tasks/get) until `status = "completed"`, then read `results[].id`.
</Note>
