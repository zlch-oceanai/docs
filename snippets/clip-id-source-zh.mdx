<Note>
**`clip_id` 从哪里获取？**

`clip_id` 标识系统中的任意音频片段。你可以从以下接口获取：

| 来源接口 | 获取方式 |
|---------|---------|
| [音乐生成](/zh/api-reference/music/generate) | 任务完成后，响应中的 `results[].id` |
| [翻唱](/zh/api-reference/music/cover) | 任务完成后，响应中的 `results[].id` |
| [Remaster](/zh/api-reference/music/remaster) | 任务完成后，响应中的 `results[].id` |
| [添加人声](/zh/api-reference/music/add-vocals) | 任务完成后，响应中的 `results[].id` |
| [添加伴奏](/zh/api-reference/music/add-instrumental) | 任务完成后，响应中的 `results[].id` |
| [哼唱生成](/zh/api-reference/music/sample) | 任务完成后，响应中的 `results[].id` |
| [音频上传](/zh/api-reference/music/upload) | `data.clipId`（同步接口，立即返回） |
| [音频拼接](/zh/api-reference/music/concat) | `data.id`（同步接口，立即返回） |
| [查询音乐 / 列表](/zh/api-reference/music/get) | 每条记录的 `id` 字段 |

对于异步生成接口，需轮询 [获取任务](/zh/api-reference/tasks/get) 直至 `status = "completed"`，然后读取 `results[].id`。
</Note>
